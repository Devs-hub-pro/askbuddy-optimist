# 后端一期 Freeze 建议说明

日期：2026-04-22  
状态：一期阶段性验证完成，当前无 P0。

## 1. Freeze 结论

建议正式进入 **Phase 1 Freeze（阶段性收官）**。  
理由：
- Pack 01 ~ Pack 08-B 主线能力已落地并完成阶段性联调验证；
- 权限边界与服务端主路径收口已建立（采纳、系统通知、订单状态流转）；
- 当前剩余问题以 P1/P2 的 cleanup 为主，不阻塞交付。

## 2. 建议的版本与发布命名

建议使用以下命名之一：

1. Git Tag（推荐）
- `backend-phase1-freeze-2026-04-22`
- `v1.0.0-phase1-freeze`

2. Release 名称（推荐）
- `AskBuddy Backend Phase1 Freeze`
- `Phase1 Backend Baseline (Pack01-08B)`

## 3. Freeze 后的变更准入规则

Freeze 后仅接受以下类型变更：
- P0/P1 bugfix（主链路错误、权限错误、数据一致性错误）
- 小型 patch migration（可审阅、可回滚、低风险）
- 文档补充与运维脚本增强
- 小范围 cleanup（不改业务模型）

Freeze 后不再接受：
- 新 Pack
- 新业务域扩展
- 大规模 schema 重构
- 大规模前端/查询层重构

## 4. Freeze 后的节奏建议

1. 进入 1~2 周稳定观察窗口（dev/staging + 小流量验证）。  
2. 观察 legacy 兼容层命中情况（follows/posts/旧命名）。  
3. 所有收尾工作走小任务单，按“可回滚、低风险”执行。  
4. 以“线上稳定性优先”作为唯一决策标准。

## 5. 交付判定

满足以下条件即可视为一期可交付：
- 当前主链路可跑通（提问/回答/采纳、搜索/发现、消息/通知、订单/账务、审核/推荐位）；
- 无 P0；
- 迁移链可复现；
- 回滚策略与上线检查清单完整。
