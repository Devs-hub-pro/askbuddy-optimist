# Phase 1 上线前角色分工建议

日期：2026-04-22  
适用范围：Phase 1 Freeze 后的小范围试运行 / 上线准备

## 1. 环境与迁移负责人

职责：
- 维护 dev/staging/prod 环境配置一致性。
- 执行并核验 `migration list` / `db push`。
- 维护迁移执行记录与回滚预案。

交付物：
- 每日迁移状态确认记录。
- 变更窗口执行结果与异常说明。

## 2. UAT 执行负责人

职责：
- 按 `backend-phase1-uat-runbook.md` 执行主链路验证。
- 跟踪通过率、失败率、复现质量。
- 将结果沉淀到 UAT 报告。

交付物：
- 每轮 UAT 执行报告。
- 主链路通过/失败明细。

## 3. 问题分诊负责人

职责：
- 将问题统一分级为 Blocker / Non-blocker。
- Blocker 触发最小 bugfix 流程。
- Non-blocker 进入 Phase 1.1 cleanup 池。

交付物：
- 问题分诊清单（含优先级与处理建议）。
- 每日问题状态看板（新增/关闭/遗留）。

## 4. service-role / 运维操作负责人

职责：
- 执行所有 service-role 受限操作（含关键 RPC 运维动作）。
- 审核并留痕高权限操作，避免误用。
- 对关键 RPC 进行定期抽检。

交付物：
- 高权限操作日志。
- 关键 RPC 安全边界抽检记录。

## 5. 试运行观测负责人

职责：
- 监控主链路稳定性与兼容层命中趋势。
- 识别潜在回归（尤其 legacy fallback、权限边界、账务链路）。
- 形成“可上线 / 暂缓上线”的观察结论。

交付物：
- 每日观测摘要。
- 周度风险汇总与建议。

## 6. 协同机制（建议）

- 每日固定 15 分钟站会：
  - 前一天 Blocker 处置结果
  - 当日高风险项
  - 是否触发回滚/暂停发布
- 统一变更规则：
  - 不开新 Pack
  - 不扩功能
  - 仅 bugfix / 小 patch / 小 cleanup
